# WB Dashboard

Адаптивный сайт-дашборд на React + Vite для продавца Wildberries.

Что показывает сайт:

- главный дашборд с общей статистикой
- показатели: продажи, комиссия WB, логистика, хранение, приёмка, штрафы, доплаты, удержания, итог
- количество заказов и количество проданных товаров
- таблицу всех товаров
- статистику за весь доступный период API WB
- фильтры по дате, товару и быстрые диапазоны: сегодня / 7 дней / 30 дней / всё

## Технологии

- React
- Vite
- Wildberries API
- Без базы данных

## Настройка

Создайте `.env` на основе `.env.example` и укажите:

```env
VITE_WB_API_TOKEN=ваш_wildberries_api_token
VITE_WB_API_BASE_URL=/wb-api
```

`VITE_WB_API_BASE_URL=/wb-api` используется для локальной разработки через Vite proxy.
В production-сборке без этой переменной проект по умолчанию ходит в `https://statistics-api.wildberries.ru`.

## Запуск сайта

1. Установите зависимости:

```bash
npm install
```

2. Запустите dev-сервер:

```bash
npm run dev
```

3. Откройте адрес, который покажет Vite, обычно:

```text
http://localhost:5173
```

## Сборка

```bash
npm run build
```

## Важный нюанс

В этом проекте токен WB используется на фронтенде через `.env`, как вы и попросили. Для локального использования это удобно, но для публичного продакшн-размещения такой подход небезопасен, потому что токен продавца нельзя считать секретным в браузере. Для продакшна нужен серверный прокси.

## Быстрые диапазоны

- `Сегодня` - текущий день
- `7 дней` - последние 7 дней
- `30 дней` - последние 30 дней
- `Всё` - весь доступный период API WB

## Фильтры

- дата от
- дата до
- товар
